## Overview

This quickstart guide will walk you through training a baseline model, running compression experiments, and exporting to ONNX for deployment. The workflow uses the built-in digits dataset for fully offline operation.

<Steps>

### Install Dependencies

First, install the required Python packages:

```bash
python -m pip install -r requirements.txt
```

### Train Your First Model

Train a k-nearest neighbors (KNN) classifier with compression and quantization experiments:

```bash
python -m research.train_cli --model knn --dataset digits --run-compression --run-quantization
```

This command will:
- Train a KNN model on the digits dataset
- Run compression experiments with default pruning levels
- Run quantization simulations (float16 and int8_sim)
- Output accuracy metrics and compression results
- Save artifacts to the `artifacts/` directory

<Note>
The `--dataset digits` flag uses scikit-learn's built-in digits dataset, which works completely offline without network access.
</Note>

### Run Benchmarking Experiments

Evaluate model performance under hardware constraints:

```bash
python -m research.evaluate_cli --model rf --dataset digits --runs 5 --run-model-compression --memory-budget-mb 8 --compute-scale 0.75
```

This performs:
- 5 repeated training runs with statistical summaries
- Model-level compression experiments at multiple levels
- Hardware simulation with 8MB memory budget and 0.75 compute scaling
- Generates confidence intervals for accuracy metrics

### Export to ONNX

Export your model for deployment with parity validation:

```bash
python -m research.infer_cli --model knn --dataset digits --batch-size 64 --export-onnx --onnx-min-agreement 0.98
```

This will:
- Train a KNN model
- Run batched CPU inference
- Export to ONNX format
- Validate that ONNX predictions match scikit-learn with ≥98% agreement
- Fail fast if parity drops below threshold

</Steps>

## Understanding the Results

All experimental outputs are saved to the `artifacts/` directory:

- **Run metadata**: JSON files with training configuration and seeds
- **ONNX models**: Exported `.onnx` files ready for deployment
- **Parity reports**: `onnx_report_*.json` files with agreement metrics
- **Compression reports**: Results from pruning and quantization experiments

<CodeGroup>

```bash Training Output
model=knn accuracy=0.983333
compression={'weight_pruning': {'accuracy': 0.976667, 'delta': -0.006666}, 'neuron_pruning': {'accuracy': 0.980000, 'delta': -0.003333}}
quantization={'float16': {'accuracy': 0.983333, 'delta': 0.000000}, 'int8_sim': {'accuracy': 0.980000, 'delta': -0.003333}}
```

```bash ONNX Export Output
batch_predictions=1797
onnx_report={'agreement': 0.9945, 'parity_pass': True, 'samples_tested': 1797}
```

</CodeGroup>

## Common Workflows

### Offline-Only Operation

For environments without network access, always use the `digits` dataset:

```bash
python -m research.train_cli --model rf --dataset digits
```

### Custom Training Parameters

Adjust sample size, seed, and pruning levels:

```bash
python -m research.train_cli \
  --model knn \
  --dataset digits \
  --sample-size 1000 \
  --seed 42 \
  --run-compression \
  --weight-prune 0.3 \
  --neuron-prune 0.15
```

### Hardware-Constrained Benchmarking

Simulate resource-constrained environments:

```bash
python -m research.evaluate_cli \
  --model knn \
  --dataset digits \
  --runs 10 \
  --memory-budget-mb 4 \
  --compute-scale 0.5
```

### Pruning Efficiency Analysis

Evaluate accuracy vs. sparsity trade-offs:

```bash
python -m research.evaluate_cli \
  --model rf \
  --dataset digits \
  --run-pruning-efficiency \
  --pruning-type weight \
  --sparsity-levels "0.0,0.2,0.4,0.6,0.8" \
  --runs 5
```
