## Overview

Reproducibility ensures that experiments produce identical results across runs, machines, and time. This framework implements multiple layers of determinism to enable reliable research and debugging.

## Explicit Seeding Strategy

### Core Implementation

The `set_deterministic` function establishes a deterministic environment:

```python
def set_deterministic(seed: int) -> None:
    os.environ["PYTHONHASHSEED"] = str(seed)
    os.environ["TF_DETERMINISTIC_OPS"] = "1"
    random.seed(seed)
    np.random.seed(seed)
```

From `reproducibility.py:10-14`.

### What Gets Seeded

<Tabs>
  <Tab title="Python Standard Library">
    ```python
    random.seed(seed)
    ```
    Controls `random.choice()`, `random.shuffle()`, `random.randint()`, etc.
  </Tab>
  <Tab title="NumPy">
    ```python
    np.random.seed(seed)
    ```
    Controls `np.random.randn()`, `np.random.choice()`, array sampling.
  </Tab>
  <Tab title="Hash Randomization">
    ```python
    os.environ["PYTHONHASHSEED"] = str(seed)
    ```
    Ensures dictionary/set iteration order is deterministic across Python processes.
  </Tab>
  <Tab title="TensorFlow (if used)">
    ```python
    os.environ["TF_DETERMINISTIC_OPS"] = "1"
    ```
    Forces TensorFlow operations to use deterministic algorithms.
  </Tab>
</Tabs>

<Warning>
**Critical**: `set_deterministic()` must be called **before** any random operations, ideally at the start of each experiment run.
</Warning>


## Deterministic Splits

### Train-Test Splitting

Scikit-learn's `train_test_split` respects the numpy random seed:

```python
from sklearn.model_selection import train_test_split

set_deterministic(seed=40)
x_train, x_test, y_train, y_test = train_test_split(
    x, y, 
    test_size=0.3,
    random_state=40,  # Must match seed for reproducibility
    stratify=y         # Preserve class distribution
)
```

<Info>
The `random_state` parameter in scikit-learn functions should match your global seed to maintain consistency.
</Info>

### Dataset Sampling

When reducing sample size:

## Comparing Experiment Runs

### Seed Schedules for Multiple Runs

Instead of a single seed, use a deterministic schedule:

```python
def repeated_benchmark(system_config: SystemConfig, training_config: TrainingConfig, runs: int = 5):
    seeds = []
    for i in range(runs):
        seed = system_config.seed + i
        seeds.append(seed)
        set_deterministic(seed)
        # Run experiment...
    
    return {"seed_schedule": seeds, ...}
```

Compute confidence intervals across runs:

```python
from research.core.utils.metrics import confidence_interval

def _summary(values: Sequence[float], z: float) -> Dict[str, float]:
    mean, std, ci = confidence_interval(values, z)
    return {"mean": mean, "std": std, "ci95": ci}
```

## Real Examples from reproducibility.py

### Example 1: Full Deterministic Setup

```python
from research.core.utils.reproducibility import set_deterministic, runtime_manifest
from research.config import SystemConfig
from research.core.data.dataset import load_dataset, split_and_normalize
from compression.classical import build_model

# Configure
config = SystemConfig(seed=40, sample_size=6000, test_size=0.3)

# Establish determinism
set_deterministic(config.seed)
manifest = runtime_manifest(config.seed)

# Load and split (now deterministic)
x, y, metadata = load_dataset(config)
x_train, x_test, y_train, y_test = split_and_normalize(x, y, config)

# Train (scikit-learn respects numpy seed)
model = build_model(TrainingConfig(model_name="knn", rf_random_state=config.seed))
model.fit(x_train, y_train)

# Evaluate
preds = model.predict(x_test)

# Save with manifest
result = {
    "manifest": manifest,
    "accuracy": float(accuracy_score(y_test, preds)),
    "test_samples": len(x_test)
}
save_json(result, config.artifacts_dir / "experiment.json")
```

## Reproducibility Checklist

<Steps>
  <Step title="Set Global Seed">
    Call `set_deterministic(seed)` at the start of every experiment.
  </Step>
  <Step title="Configure Model Seeds">
    Pass `random_state` or equivalent to all scikit-learn estimators:
    ```python
    TrainingConfig(rf_random_state=40)
    ```
  </Step>
  <Step title="Use Stratified Splits">
    Enable `stratify=y` in `train_test_split` to preserve class distributions.
  </Step>
  <Step title="Record Metadata">
    Capture `runtime_manifest()` and `seed_schedule` in all reports.
  </Step>
  <Step title="Version Artifacts">
    Include `summary_version` field to track report schema changes.
  </Step>
  <Step title="Compute Confidence Intervals">
    Run experiments multiple times (3-10 runs) and report mean ± CI.
  </Step>
</Steps>

## Limitations

From `hardware_profiling.md:59-64`:

> When reporting results:
> 1. record exact CLI command,
> 2. record seed and dataset choice,
> 3. include confidence intervals across repeated runs,
> 4. retain generated JSON and plot artifacts.

<Warning>
Deterministic settings can mask variance that appears in uncontrolled production settings (from `system_design.md:27`).
</Warning>

### Non-Deterministic Factors

- **Hardware timing**: CPU scheduling and cache states introduce minor latency variance
- **Network I/O**: Dataset downloads may vary in timing
- **Floating-point non-associativity**: Some operations have platform-dependent rounding
- **Multi-threading**: Parallel execution can introduce ordering non-determinism

## Best Practices

1. **Use consistent seeds**: Default to `seed=40` across all experiments
2. **Run multiple times**: 5-10 runs for statistical robustness
3. **Archive artifacts**: Store JSON reports and plots in version control or artifact stores
4. **Document environment**: Include Python version, OS, and library versions
5. **Test reproducibility**: Re-run experiments on different machines to verify consistency
