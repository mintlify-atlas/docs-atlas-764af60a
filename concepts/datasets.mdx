## Dataset Loading

### Load Function

The `load_dataset` function handles both datasets with automatic fallback:

```python research/core/data/dataset.py
def load_dataset(config: SystemConfig):
    if config.dataset == "mnist":
        try:
            (x_train, y_train), _ = tf.keras.datasets.mnist.load_data()
            x = x_train.reshape(x_train.shape[0], 28 * 28).astype(np.float32)
            y = y_train.astype(np.int64)
            version = tf.__version__
        except Exception as exc:
            if config.fail_fast_dataset:
                raise RuntimeError("MNIST download/load failed with fail-fast enabled") from exc
            raise RuntimeError(
                "MNIST download/load failed. Use --dataset digits for offline runs or --fail-fast-dataset for strict mode."
            ) from exc
    elif config.dataset == "digits":
        ds = load_digits()
        x = ds.data.astype(np.float32)
        y = ds.target.astype(np.int64)
        version = sklearn.__version__
    else:
        raise ValueError(f"Unsupported dataset={config.dataset}")

    n = min(config.sample_size, len(x))
    x, y = x[:n], y[:n]
    metadata = {
        "dataset": config.dataset,
        "version": version,
        "checksum_sha256": _dataset_checksum(x, y),
        "source_samples": int(len(x)),
    }
    return x, y, metadata
```

### Configuration

```python research/config.py
@dataclass
class SystemConfig:
    seed: int = 40
    sample_size: int = 6000
    test_size: float = 0.3
    normalize: bool = True
    dataset: str = "mnist"  # or "digits"
    fail_fast_dataset: bool = False
    artifacts_dir: Path = Path("artifacts")
    confidence_z: float = 1.96
```

<ParamField path="dataset" type="str" default="mnist">
  Which dataset to load:
  - `"mnist"`: TensorFlow MNIST (28×28, 70k samples, network required)
  - `"digits"`: scikit-learn digits (8×8, 1797 samples, offline)
</ParamField>

<ParamField path="sample_size" type="int" default={6000}>
  Number of samples to use from the dataset. Capped at dataset size.
</ParamField>

<ParamField path="fail_fast_dataset" type="bool" default={false}>
  If `True`, raise error immediately if MNIST download fails. If `False`, show helpful error message suggesting `--dataset digits`.
</ParamField>

## Dataset Fingerprinting

### SHA-256 Checksums

Every loaded dataset is fingerprinted with a SHA-256 checksum for reproducibility:

```python research/core/data/dataset.py
def _dataset_checksum(x: np.ndarray, y: np.ndarray) -> str:
    hasher = hashlib.sha256()
    hasher.update(np.ascontiguousarray(x).tobytes())
    hasher.update(np.ascontiguousarray(y).tobytes())
    return hasher.hexdigest()
```

The checksum is computed over:
- Feature matrix `x` (samples × features)
- Label vector `y` (samples,)

### Fingerprint Metadata

```python research/core/data/dataset.py
def dataset_fingerprint(x: np.ndarray, y: np.ndarray, metadata: Dict) -> Dict:
    return {
        "shape": list(x.shape),
        "labels": sorted(list(map(int, np.unique(y)))),
        "dtype": str(x.dtype),
        "dataset": metadata["dataset"],
        "version": metadata["version"],
        "checksum_sha256": metadata["checksum_sha256"],
    }
```

Fingerprints include:
- `shape`: Array dimensions (samples, features)
- `labels`: Unique class labels in sorted order
- `dtype`: Data type (typically `float32`)
- `dataset`: Dataset name (`"mnist"` or `"digits"`)
- `version`: TensorFlow or scikit-learn version
- `checksum_sha256`: SHA-256 hash of data

### Artifact Persistence

Fingerprints are saved to JSON for reproducibility verification:

```python research/core/data/dataset.py
def save_dataset_metadata(config: SystemConfig, x: np.ndarray, y: np.ndarray, metadata: Dict) -> Path:
    path = config.artifacts_dir / "dataset_metadata.json"
    payload = {"config": asdict(config), "fingerprint": dataset_fingerprint(x, y, metadata)}
    payload["config"]["artifacts_dir"] = str(config.artifacts_dir)
    save_json(payload, path)
    return path
```

<Note>
Checksum verification ensures identical input data across runs and detects any data corruption or version mismatches.
</Note>

## Train/Test Splitting

### Stratified Split

The system uses stratified train/test splitting to preserve class distributions:

```python research/core/data/dataset.py
def split_and_normalize(x: np.ndarray, y: np.ndarray, config: SystemConfig) -> Tuple[np.ndarray, ...]:
    x_train, x_test, y_train, y_test = train_test_split(
        x, y, test_size=config.test_size, random_state=config.seed, stratify=y
    )
    if config.normalize:
        normalizer = Normalizer()
        x_train = normalizer.fit_transform(x_train)
        x_test = normalizer.transform(x_test)
    return x_train, x_test, y_train, y_test
```

### Split Parameters

<ParamField path="test_size" type="float" default={0.3}>
  Fraction of data reserved for testing. Default 30% test, 70% train.
</ParamField>

<ParamField path="seed" type="int" default={40}>
  Random seed for reproducible splitting. Same seed produces identical splits.
</ParamField>

<ParamField path="stratify" type="array">
  Class labels for stratified splitting. Ensures each split has proportional class representation.
</ParamField>

### Example Split

With `sample_size=6000` and `test_size=0.3`:

```
Total samples:    6000
Train samples:    4200 (70%)
Test samples:     1800 (30%)

Class distribution preserved:
- Class 0: ~600 total → ~420 train, ~180 test
- Class 1: ~600 total → ~420 train, ~180 test
...
```

<Check>
**Stratification ensures balanced evaluation** even with imbalanced classes.
</Check>

## Normalization

### L2 Normalization

Features are normalized using L2 norm (Euclidean norm) to unit length:

```python research/core/data/dataset.py
if config.normalize:
    normalizer = Normalizer()  # L2 norm by default
    x_train = normalizer.fit_transform(x_train)
    x_test = normalizer.transform(x_test)
```

### Why L2 Normalization?

<AccordionGroup>
  <Accordion title="Distance Metrics">
    KNN uses distance metrics (Euclidean, Manhattan) that are sensitive to feature scale. L2 normalization ensures all samples have equal magnitude.
  </Accordion>
  
  <Accordion title="Numerical Stability">
    Prevents overflow/underflow in distance calculations and improves numerical precision.
  </Accordion>
  
  <Accordion title="Model Convergence">
    Random Forest benefits from normalized features for better split quality and tree balance.
  </Accordion>
</AccordionGroup>

### Normalization Formula

For each sample vector **x**:

```
x_normalized = x / ||x||₂

where ||x||₂ = √(x₁² + x₂² + ... + xₙ²)
```

<Note>
Normalizer is fit on training data only, then applied to test data to prevent data leakage.
</Note>

### Configuration

<ParamField path="normalize" type="bool" default={true}>
  Enable L2 normalization. Set to `False` to use raw pixel values.
</ParamField>

<Warning>
From `docs/system_design.md:51`: The system assumes input features are numeric and can be normalized without domain loss. Non-numeric or categorical features are not supported.
</Warning>

## Data Flow Summary

<Steps>
  <Step title="Load Dataset">
    Download or load from cache (MNIST) or use bundled data (digits)
    
    **Output**: `(x, y, metadata)` with shape `(samples, features)`
  </Step>
  
  <Step title="Fingerprint">
    Compute SHA-256 checksum and metadata
    
    **Output**: JSON artifact with checksum and version info
  </Step>
  
  <Step title="Split">
    Stratified train/test split with fixed seed
    
    **Output**: `(x_train, x_test, y_train, y_test)`
  </Step>
  
  <Step title="Normalize">
    L2 normalization fit on train, applied to test
    
    **Output**: Normalized feature matrices ready for training
  </Step>
</Steps>
